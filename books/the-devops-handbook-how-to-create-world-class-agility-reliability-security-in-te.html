Highlights (AI · MDDE/SBM lens)

AI-distilled highlights, not personal reading highlights — the ideas in this book that connect to Jaco's MDDE / Structure-Beats-Magic work, selected and paraphrased through that lens. Paraphrased (never verbatim); verbatim quotes await the book text.

  • The Three Ways: flow, feedback, continual learning — the book's spine is a systems view of delivery, and the third way (a system that learns from its own operation) is the same loop a self-improving knowledge system runs: work produces documentation, documentation improves the next round of work.
  • Everything in version control, everything reproducible — the deployment pipeline demands that environments and changes be recreatable from controlled sources on demand; MDDE is this exact argument transplanted to data, with the model as the versioned source and the pipeline as the reproducible build.
  • Small batch sizes beat heroic releases — shrinking the unit of change makes flow faster and failure cheaper, which is also why pattern-conformant models extend incrementally (one hub, one satellite) instead of via big-bang remodels.
  • Automate so the standard is the easy path — high performers encode their practices into the platform rather than relying on discipline and heroes; heroics are the "magic" that structure is supposed to replace.
  • Build quality checks into the pipeline, not after it — the andon-cord idea of stopping the line on defect detection is rules executed automatically at the moment of change, the operational form of rules as the third pillar next to data and code.
  • Make work visible — invisible WIP is unmanageable WIP; the same reason a knowledge vault indexes everything into daily notes and dashboards instead of trusting memory.
