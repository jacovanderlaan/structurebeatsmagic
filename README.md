# Structure Beats Magic — hub site

The **master-brand landing site** for the *Structure Beats Magic* thesis:
Knowledge Management, re-architected for the AI era.

> The magic isn't in the model. It's in the structure you give it.
> **Structure + Data + AI + Rules + Skills → Systems.**

## Role in the brand architecture

This is the **umbrella hub**. It explains the universal principles, shows the two
implementations (personal + business), links to live sample systems, and routes
serious organisations onward to the enterprise consultancy.

```
structurebeatsmagic.com   ← THIS SITE (master / thesis hub)
  ├─ Universal principles   (the formula, free "recipe")
  ├─ Personal implementation (proof in the life: photos, books, trips)
  ├─ Business implementation (proof in the work: governed quality for mid-size
  │     orgs — "bankers vs builders")
  ├─ Sample systems          (live demos / write-ups)
  └─ → jacovanderlaan.com    (high-value enterprise consultancy — the cash lane)
```

It deliberately gives the **what/why** free and routes the **how/method** to paid
engagements ("give the recipe, sell the kitchen").

## Stack

Static-first per ADR-046: plain HTML + a shared `assets/site.css` / `article.css`,
no framework, no runtime. The hub (`index.html`) and section pages (`/system`,
`/intelligence`, `/influences`) are hand-authored HTML. The **articles** are the
one built part: markdown drafts → `build_articles.py` → styled `articles/*.html`.
Brand palette (off-white / deep navy / blue accent / gold payoff) shared with
jacovanderlaan.com.

## Build (articles + sitemap)

The hub pages need no build. The articles do:

```bash
python build_articles.py
```

This renders the allow-listed markdown drafts (from
`W:/data/products/mdde/output/articles/drafts/`, override with `SBM_DRAFTS`) into
`articles/*.html`, writes `articles/_cards.html` (the homepage Writing strip), and
**regenerates `sitemap.xml`**. Supported markdown + the figure/code rules live in
[STYLEGUIDE.md](STYLEGUIDE.md). Run it before committing whenever a draft changes.

## Preview

Open `index.html` directly, or serve the folder:

```bash
python -m http.server 8000   # then open http://localhost:8000
```

## Deploy

GitHub Pages via Actions (`.github/workflows/deploy.yml`), publishing the repo
root on push to `main`. No build command on the host — `build_articles.py` runs
locally before commit.

Intended target domain: `structurebeatsmagic.com`. It currently **301-redirects to
jacovanderlaan.com** (interim); this site becomes the destination after the DNS
cutover at Dynadot. Post-cutover, swap `BASE_URL` in `build_articles.py` and the
`Sitemap:` line in `robots.txt` to the custom domain (see `DOMAIN.md`).

## Crawlers / SEO

`robots.txt` welcomes all crawlers, including AI ones — this is a discovery hub,
so being indexed and cited is the goal (ADR-060). `sitemap.xml` is generated by
the build. Privacy is controlled by **not publishing personal data**, not by
blocking bots — see the asset-hygiene section of [STYLEGUIDE.md](STYLEGUIDE.md).

## Assets

`assets/` is **published-only**: every file must be referenced by a `src=`/`url()`
in some page or stylesheet. SBM brand graphics (hero / formula / toy-vs-instrument
/ og-card SVGs) plus the placed article infographics. Unused/orphan images are
moved — never deleted — to private staging
(`W:/data/products/mdde/output/articles/images/sbm-unused/`). Audit before every
commit; never commit personal photos.
